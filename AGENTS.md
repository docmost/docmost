# AGENTS.md — Docmost

## Project overview

Docmost is a collaborative wiki. Monorepo with **pnpm workspaces** + **Nx** orchestration.

```
apps/server/   NestJS + Fastify (CommonJS, Jest)
apps/client/   React + Vite + Mantine UI (ESM, Vitest)
packages/ee/   Enterprise license file only (no code)
```

## Commands

```bash
pnpm dev                       # Both client + server concurrently
pnpm build                     # Full build (nx run-many)
pnpm --filter ./apps/server run start:dev
pnpm --filter ./apps/client run dev

# Testing (run from monorepo root)
pnpm --filter ./apps/server run test        # Jest: **/*.spec.ts
pnpm --filter ./apps/client run test        # Vitest

# Linting
pnpm --filter ./apps/server run lint        # ESLint --fix
pnpm --filter ./apps/client run lint

# DB migrations (server only)
pnpm --filter ./apps/server run migration:latest
pnpm --filter ./apps/server run migration:create

# Regenerate Kysely DB types after schema changes
pnpm --filter ./apps/server run migration:codegen
# → outputs to apps/server/src/database/types/db.d.ts (AUTO-GENERATED, DO NOT EDIT)
```

## Architecture

### Server (NestJS + Fastify)

- **Global prefix** `api` set in `main.ts`, with exclusions: `robots.txt`, `share/:shareId/p/:pageSlug`, `mcp`.
- **All controllers use `@Post()`** for every endpoint (RPC-style, not REST).
- **Kysely ORM** with PostgreSQL — no ActiveRecord/TypeORM. Raw SQL via query builder. Types in `apps/server/src/database/types/db.d.ts` are auto-generated by `kysely-codegen`.
- **Multi-tenant by workspace**: requests pass through a `preHandler` hook that resolves `workspaceId` from hostname. Auth is JWT-based via `passport-jwt`. The JWT payload always carries `workspaceId`.
- **Key decorators**: `@AuthUser()` extracts `{ user, workspace }` from request, `@AuthWorkspace()` extracts workspace.

### Path aliases (server tsconfig)
| Alias | Resolves to |
|---|---|
| `@docmost/db/*` | `./src/database/*` |
| `@docmost/ee/*` | `./src/ee/*` |
| `@docmost/transactional/*` | `./src/integrations/transactional/*` |

### Client (React + Vite)
- `@/*` → `./src/*`
- Mantine UI v8, Jotai for state, React Query for data fetching, React Router v7.

### DB migratons

Migrations live in `apps/server/src/database/migrations/*.ts`. They export `up(db: Kysely)` and `down(db: Kysely)`. Run via `tsx src/database/migrate.ts`.

## Enterprise edition separation

- **`apps/server/src/ee/`** — Exists but is **empty** in the public AGPL checkout. The backend loads it dynamically via `require('./ee/ee.module')?.EeModule` in `app.module.ts`. If the require fails, enterprise features are skipped gracefully (unless `CLOUD=true`, which exits with error).
- **`apps/client/src/ee/`** — Frontend enterprise UI **is present** but feature-gated via `useHasFeature()` checking the `entitlementAtom`. Features unlock when `POST /api/workspace/entitlements` returns them in the `features` array.
- **License validation** is centralized in `LicenseCheckService` (`apps/server/src/integrations/environment/license-check.service.ts`). It tries to `require('../../ee/licence/license.service')`; on failure, all features return `false`/empty.
- **Enterprise features defined** in `apps/server/src/common/features.ts` and `apps/client/src/ee/features.ts`. Keys include: `ai`, `mcp`, `api:keys`, `sso:custom`, `sso:google`, `mfa`, `scim`, `page:permissions`, etc.
- **Adding EE backend modules**: Create files in `apps/server/src/ee/` exporting an `EeModule` NestJS module. The dynamic `require()` in `app.module.ts` will pick it up automatically.

## Key invariants

- **Never edit** `apps/server/src/database/types/db.d.ts` manually — it's generated by `kysely-codegen`.
- **strictNullChecks is off** in both server and client tsconfigs. Do not assume strict null safety.
- **No `.env` committed** — copy `.env.example` and fill in values. `APP_SECRET` must be ≥32 chars and not the placeholder value.
- **Style**: server uses `@HttpCode(HttpStatus.OK)` + `@Post()` for every route. Follow existing patterns.
- **Dependencies**: PostgreSQL + Redis are required (see `docker-compose.yml`). Optional: Typesense for advanced search, pgvector for AI embeddings.
