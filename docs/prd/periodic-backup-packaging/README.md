# Docmost 定期备份打包 PRD 包

本目录用于定义 Docmost 新增“定期备份打包文件”能力的产品与交付标准，面向产品、研发、测试与运维协同。

## 文档索引

- `PRD.md`：产品需求主文档（目标、范围、需求、指标、验收）
- `TECH-SPEC.md`：技术方案与接口草案（架构、数据模型、流程、安全）
- `ROLL-OUT-AND-OPS.md`：分阶段上线计划与运维手册
- `TEST-CASES.md`：验收测试用例矩阵（功能/异常/性能/恢复）

## 适配当前项目的关键事实

- 当前核心数据在 PostgreSQL（`DATABASE_URL`），文件在 `STORAGE_DRIVER`（`local` 或 `s3`）。
- Docker 默认把本地文件存储挂载到 `/app/data/storage`。
- 代码中已有“页面/空间导出 ZIP”能力，但不等同于“实例级灾备快照”。
- 后端为 NestJS，已有 `@nestjs/schedule` 与队列能力，可承载定时备份任务。

## 本 PRD 包输出目标

- 定义一套可在 Docmost 自托管场景落地的备份产品能力。
- 对齐行业最佳实践：3-2-1、可恢复性验证、加密、最小权限、审计可追踪。
- 明确可执行的里程碑、验收标准和上线后运维要求。
